Fraud Detection System

The surge in online credit/debit card transactions has led to an upsurge in fraud. To combat this, we're employing classification algorithms like Random Forest. Using a vast dataset, we'll train and evaluate these models, selecting the one with the highest accuracy, precision, and a low false positive rate. The chosen model will be saved in pkl format. We'll integrate it into a Flask web application for real-time detection. This project aims to enhance fraud detection, save costs, and offer real-time protection.

Understanding "Step"

Step represents the time point of a transaction within a specific day in the dataset. It is essentially a timestamp index that helps us track when a particular transaction took place. The values of "step" range from 1 to 744, where:

  • Step 1: Represents the first transaction of the day (00:00 hour).
  • Step 744: Represents the last transaction of the day (23:59 hour).
  • The "step" value is essentially a time encoding, where each value corresponds to an hour in the day. For example, step 1 would indicate the start of the day, and as the step value increases, it moves toward the end of the day.
  • Since there are 24 hours in a day, step values are calculated to fit into this timeframe, and the model uses these values to understand the distribution of transactions throughout the day.

This helps us understand patterns like peak transaction times or any unusual behavior in transaction activities, which could indicate fraud.

Understanding Transaction Types

The dataset contains several transaction types, each represented by a unique number:

  • Type 0: Payment - These transactions are usually payments to merchants or bill payments.
  • Type 1: Transfer - Fraud is most likely to occur in these transactions, as they involve transferring funds to another account.
  • Type 2: Cash-in - Adding funds to the account, which are generally non-fraudulent transactions.
  • Type 3: Deposit - These transactions typically involve deposits from other accounts or services.
  • Type 4: Cashout - Fraud is highly likely in cashout transactions, where funds are withdrawn from the account.

Fraudulent activities are observed primarily in Transfer (Type 1) and Cashout (Type 4) transactions based on the exploratory data analysis (EDA) of the dataset.
